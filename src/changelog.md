## Версия 1.3.2 (2026-04-28)
- добавлен контроль занимаемого логами дискового пространства
- доработка режима калибровки:
  - азимут корректируется согласно углу поворотного устройства при помощи LHOV 
  - при помощи команды LHOV задается эмуляция координат при старте калибровки
  - координаты по умолчанию для режима калибровки задаются командой SCLHOV (командная строка)
  - технологический режим управляется параметров командной строки SEAR,порт_Радант,скорость_порта
  - запуск калибровки: SCAL,startAngle,step,count
  - остановка калибровки: FCAL
  - статус калибровки: CAL? -> CAL,state,collectedPoints
- мелкие исправления интерфейса
- добавлен режим совместимости с протоколом SIMRAD (команда PSIMSSB)
- добавлена возможность задания скорости звука (команда SET2). Планируется расширение списка параметров этой команды 
  она будет использоваться для задания доплнительных настроек в будущем

## Версия 1.0.5. (2026-02-17)

- Возможность задания географических координат опорных маяков-ответчиков и получение абсолютных собственных координат
  команда SRC3 теперь поддерживает географические коордианты (первый параметр = 2). Обработка происходит в 
  AZMCombainer.Set3RespondersGeographicCoordinates
- команда FLTS позволяет настроить параметры фильтрации для LBL и USBL режимов
- команда SETS выводит текущие настройки приложения
- команда SIOC позволяет задать индивидуальные парметры UDP-соединения для каждого маяка-ответчика. По этому соединению
  передаются навигационные данные, относящиеся к этому маяку в формате NMEA0183 (GGA, RMC, WTM)
  
- Обширный рефакторинг
- перевод на интерполяцию строк почти везде
- переход на async await при обработке команд терминала
- словарь с маяками теперь потокобезопасный (ConcurrentDictionary)
